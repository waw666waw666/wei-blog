## 背景

PakePlus 的目标是让 Web 项目能够以极小的体积打包为桌面和移动端应用。

## 技术方案选型

### 桌面端

| 方案 | 体积 | 性能 | 开发成本 |
|------|------|------|---------|
| Electron | 100MB+ | 中 | 低 |
| Tauri | 5MB | 高 | 中 |
| PakePlus | <5MB | 高 | 低 |

PakePlus 选择了类似 Tauri 的轻量方案，使用系统 WebView 代替 Chromium。

### 移动端

| 平台 | 方案 |
|------|------|
| Android | WebView + Lua 桥接 |
| iOS | WKWebView + Swift 桥接 |

## 实现原理

### 核心思路
1. 使用系统原生 WebView 渲染页面
2. 通过桥接层实现原生 API 调用
3. 打包时只包含桥接代码和 Web 资源

### 体积优化
- 不打包 Chromium（节省 100MB+）
- 精简桥接层代码
- 按平台编译，无冗余代码

## 兼容性

支持的主流前端框架：
- 纯 HTML/CSS/JS
- Vue / React / Svelte
- 任意静态网站

## 使用流程

```
1. 准备你的 Web 项目
2. 使用 PakePlus CLI 指定入口 URL
3. 选择目标平台
4. 一键打包输出安装包
```

## 适用场景

- 工具类网站转为桌面工具
- 管理后台转为移动 App
- 个人项目打包发布到应用商店
