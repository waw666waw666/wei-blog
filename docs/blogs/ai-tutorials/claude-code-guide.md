# Claude Code 从入门到精通

用了几个月 Claude Code，从一个只会敲几个命令的小白到现在能用它完成复杂任务，这中间踩了不少坑，也总结出了一些自己的心得。这篇东西就当是一个使用笔记吧，希望能帮你少走弯路。

## 什么是 Claude Code

简单说，Claude Code 是一个终端里的 AI 编程助手。它不是那种在 IDE 里帮你补全代码的插件，而是在命令行里跟你对话，理解你的需求，然后帮你写代码、改代码、查问题。

和 Cursor、Copilot 最大的区别是：Claude Code 是**以对话为中心**的。你可以跟它说"帮我重构这个模块"，它真的会去读你的代码，理解整个项目结构，然后给你一个完整的方案。

## 安装

```bash
npm install -g @anthropic-ai/claude-code
```

安装完跑 `claude` 就能启动。第一次会要你输入 API Key，去 [console.anthropic.com](https://console.anthropic.com) 申请就行。

## 我的常用场景

### 场景一：代码审查

这是我用得最多的功能。写完代码跑一遍 `claude`，然后说：

```
帮我 review 一下最近的改动，关注性能和安全问题
```

它会自动读 diff，然后逐行指出问题。有一次它发现了我没注意到的 SQL 注入风险——虽然只是个内部工具，但这个习惯后来救了我好几次。

### 场景二：重构

重构老代码是最头疼的事。Claude Code 能做这个：

```
把这个 Express 路由改造成 TypeScript，加上类型定义
```

它会读整个文件，理解逻辑，然后输出改造后的代码。但**一定要 review 它的输出**，有时候它会改掉你不想改的逻辑。

### 场景三：写测试

没人喜欢写测试，我也不例外。现在我都是先写好功能代码，然后让 Claude Code 帮我生成测试：

```
给这个模块写单元测试，覆盖正常情况和边界情况，用 Jest
```

生成完后我再看一遍，调整一下边界条件。效率提升明显。

### 场景四：调试

遇到 bug 的时候，把错误信息复制进去问它。它比搜索引擎好使的地方是——它能看到你的代码上下文，不需要你手动贴一堆文件。

## 进阶技巧

### 1. 用 /compact 模式省 Token

如果你在用付费 API，Token 消耗是个现实问题。`/compact` 模式会让 Claude Code 的回答更精简，省 Token 效果显著。

### 2. 善用 CLAUDE.md

项目根目录放一个 `CLAUDE.md`，写上项目描述、技术栈、编码规范。然后每次启动 Claude Code 它会自动读取，响应会更准确。

我的 `CLAUDE.md` 长这样：

```markdown
# 项目概述
XX 管理系统，后端 Express + TypeScript，前端 React + Vite

# 编码规范
- 使用 2 空格缩进
- 函数命名使用 camelCase
- 组件使用 PascalCase
- 所有 API 路由放在 routes/ 目录下
```

### 3. 分步骤提问

不要把啥都塞到一个问题里。先让它理解项目结构，再给具体任务。就像带新人一样，一步一步来效果最好。

## 踩过的坑

### 一次删掉了我刚写的代码
有次我说"重构这个文件"，它理解错了意思，直接把整个文件内容替换了。还好有 git，`git checkout` 救回来了。**重要操作前记得先提交。**

### 上下文窗口满了就不认账
长对话里 Claude Code 会忘记前面说过的话。这时候用 `/clear` 清空对话重新开始，但要把关键上下文重新描述一遍。

### API 不稳定
有几次遇到 API 超时，特别是代码量大的时候。后来我学会了分文件处理，不要一次让它处理整个项目。

## 总结

Claude Code 不是万能药，但用好了确实能大幅提升效率。我的经验是：把它当成一个**能力很强但需要监督的实习生**，大胆用，但要仔细 review 它的输出。

最后分享一个我觉得最有用的 workflow：写代码前先跟它讨论方案，理清思路再动手。很多时候讨论完你自己就知道怎么写了，它就像是一个能随时对话的技术合伙人。
