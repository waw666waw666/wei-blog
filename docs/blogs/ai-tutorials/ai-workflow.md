# 搭建个人 AI 工作流：我的日常工具链

从去年开始，我陆续把 AI 工具整合到了日常工作流里。现在回过头来看，效率提升是实打实的。分享一下我目前的配置，和一些踩坑经验。

## 我的工具矩阵

| 场景 | 工具 | 用途 |
|------|------|------|
| 日常编程 | Claude Code | 写代码、调试、重构 |
| 快速问答 | ChatGPT / Claude | 查资料、 brainstorm |
| 翻译 | DeepSeek | 技术文档翻译 |
| 图片生成 | Midjourney | 配图、素材 |
| 内容总结 | AI-tg-radar-bot | RSS 资讯推送 |

## 工作流一：每日 AI 情报获取

我自己写了一个 AI 情报推送机器人，流程是这样的：

```
RSS 源 → 自动抓取 → DeepSeek 摘要 → 推送到 Telegram
```

每天早上打开 TG 就能看到精选的 AI 资讯，每篇都带摘要和原文链接。不用再手动刷新闻了。

订阅的源包括：
- OpenAI / Anthropic / Google AI 官方博客
- arXiv 的 AI 分类
- Hacker News 的机器学习板块
- 几个中文 AI 资讯站

## 工作流二：AI 辅助编程

我的编程工作流分三个阶段：

### 阶段 1：方案讨论
在写代码之前，先跟 Claude Code 讨论技术方案。

```
我想实现一个文本水波纹效果，类似石子扔进水里那种。
有哪几种方案？各自的优缺点是什么？
```

AI 会给出几种方案，然后我选一个最适合的。这个阶段帮我避免了很多"写到一半才发现方案不行"的情况。

### 阶段 2：代码生成
方案确定后，让 AI 生成核心代码。

```
选方案 2（Canvas + 波动方程），帮我实现基础框架。
关键要求：
- 60fps 渲染
- 鼠标交互作为波源
- 阻尼衰减
```

### 阶段 3：Review + 测试
AI 生成的代码我不会直接用。先 review 一遍，然后让 AI 补充测试。

### 用 AI 写 commit message
这是我觉得最实用的小技巧之一：

```bash
claude "帮我写一个 commit message"
```

它会读 diff，自动生成规范的 commit message。省掉了写 commit message 这个最烦人的步骤。

## 工作流三：AI 辅助写作

写技术文章的时候，我用 AI 来处理这些事情：
- **润色** — 写完一段话让 AI 改得更通顺
- **补充细节** — 觉得哪里说得不够清楚，让 AI 帮忙展开
- **生成示例代码** — 文章的代码片段可以让 AI 生成
- **翻译** — 需要翻译英文资料时，DeepSeek 的翻译效果个人感觉还不错

## 工作流四：自动化脚本

日常有很多小任务可以用 AI 脚本来完成：

1. **批量重命名文件** — 描述需求，AI 生成 PowerShell 脚本
2. **日志分析** — 把日志喂给 AI，让它找出异常模式
3. **数据转换** — 从 JSON 转 CSV、从 Markdown 转 HTML 等
4. **图片压缩** — 让 AI 写批量处理脚本

## 省钱技巧

### API 费用控制
CLI 工具按 Token 计费，用得多了费用确实可观。几个省钱方法：

1. **本地模型兜底** — 简单任务用本地 Ollama 跑，复杂任务才调 API
2. **缓存重复请求** — 相同的代码审查请求结果可以缓存
3. **控制上下文** — 不必要的上下文不要塞给 AI

## 写在最后

AI 工具再好也是工具，关键还是看你怎么用。我的建议是：

1. **不要一次性颠覆工作流** — 从小处开始，一个场景一个场景地引入
2. **保持批判性思维** — AI 会犯错，尤其是涉及具体业务逻辑的时候
3. **建立自己的模板库** — 常用的 Prompt 存起来，反复用

工具在变，但解决问题的思路不会变。把 AI 当成你的得力助手，而不是替代品，这才是正确的打开方式。
