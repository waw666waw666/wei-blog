## 项目背景

前端组件展示平台的目标是打造一个组件"百科全书"——每个组件独立可运行，附带完整代码示例，方便开发者查阅和复用。

## 架构设计

### 组件分类体系

| 分类 | 数量 | 代表组件 |
|------|------|---------|
| 通用 | 10+ | Button、Input、Select |
| 反馈 | 8+ | Modal、Drawer、Notification |
| 数据展示 | 12+ | Table、Tree、Card |
| 拖拽排序 | 6+ | Sortable List、Drag Grid |
| 导航 | 5+ | Menu、Tabs、Breadcrumb |
| AI 模板 | 10+ | AI 提示词模板 |

### 代码组织

每个组件包含：
- 演示文件（展示效果）
- 源代码（完整可运行）
- 文档说明（使用方式）
- API 表格（属性说明）

## 技术实现

### 按需加载
使用 React.lazy + Suspense 实现组件的按需加载，首页秒开。

### 代码展示
自定义代码高亮组件，支持一键复制和展开/收起。

### AI 模板
结合 AI 生成组件示例代码，用户输入需求即可生成对应的组件使用示例。

## 设计模式

### 容器-展示模式
每个组件分为容器层（数据逻辑）和展示层（UI 渲染），职责清晰。

### Hooks 抽象
通用逻辑（如拖拽、搜索、分页）封装为自定义 Hooks，跨组件复用。
