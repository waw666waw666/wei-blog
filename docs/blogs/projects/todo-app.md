## 为什么做待办应用？

市面上的待办应用普遍存在两个问题：
1. 功能臃肿 — 一个待办应用塞了日历、笔记、项目管理
2. 体积庞大 — 动辄几十 MB，启动慢

ToDo 极简待办的核心理念：**只做一件事，做到极致。**

## 设计取舍

### 不做
- 云端同步（增加复杂度）
- 日历视图（偏离核心场景）
- 团队协作（引入账户系统）
- 通知推送（需要有后台服务）

### 只做
- 添加/编辑/删除待办
- 拖拽排序
- 分类标签
- 本地持久化

## 技术实现

### 数据存储
使用 localStorage + 版本控制，确保数据不会因为应用更新而丢失。

### 性能优化
3MB 的体积来自：
- 零外部依赖
- 精简的 CSS
- 手写优化后的 JavaScript

## 设计风格

高颜值来自对细节的打磨：
- 精心调配的颜色系统
- 流畅的过渡动画
- 合理的间距和留白
- 一致的圆角体系

## 总结

做减法比做加法难。每一个"不做"的决定背后，都是对产品本质的思考。
